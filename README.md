# Планировщик задач Mini Cron

Консольная программа на C++, которая позволяет создавать расписание запуска процессов и файлов. Программа похожа на `cron` в Linux: пользователь добавляет задачи, указывает время запуска, а затем включает режим планировщика.

## Возможности

- добавление задач с названием и командой запуска;
- запуск задачи один раз в конкретную дату и время;
- ежедневный запуск задачи в указанное время;
- повторный запуск задачи через заданный интервал в минутах;
- просмотр списка задач;
- удаление задач по ID;
- включение и выключение задач;
- сохранение задач в файл `tasks.txt`;
- автоматический запуск команд, когда наступает нужное время.

## Сборка

Для сборки нужен компилятор C++ с поддержкой C++17, например `g++`.

```powershell
cd C:\Users\AKSI\Desktop\Progs\RSM_prj
g++ RSMpraktika.cpp -std=c++17 -o RSMpraktika.exe
```

## Запуск

```powershell
.\RSMpraktika.exe
```

После запуска появляется меню:

```text
=== Mini Cron ===
1. Показать задачи
2. Добавить задачу
3. Удалить задачу
4. Включить/выключить задачу
5. Запустить планировщик
0. Выход
```

## Как пользоваться

### 1. Добавление задачи

Выберите пункт `2. Добавить задачу`.

Программа попросит ввести:

- название задачи;
- команду или путь к файлу;
- тип расписания.

Доступные типы расписания:

```text
1. Один раз в конкретную дату и время
2. Каждый день в указанное время
3. Каждые N минут
```

### 2. Запуск планировщика

После добавления задач нужно выбрать пункт:

```text
5. Запустить планировщик
```

После этого программа начинает проверять задачи каждые 5 секунд. Если текущее время стало больше или равно времени запуска задачи, программа выполняет команду.

Остановить работающий планировщик можно сочетанием клавиш: Ctrl + C

## Примеры работы

### Пример 1. Запустить Блокнот один раз

```text
Название задачи: Блокнот
Команда для запуска: notepad.exe
Тип расписания: 1
Дата и время: 2026-05-22 18:30
```

В указанное время программа запустит `notepad.exe`. После запуска одноразовая задача автоматически выключится.

### Пример 2. Запускать Калькулятор каждый день

```text
Название задачи: Калькулятор
Команда для запуска: "C:\Users\AKSI\Desktop\calc.exe"
Тип расписания: 2
Время каждого дня: 09:00
```

Каждый день в 09:00 планировщик будет запускать калькулятор.

### Пример 3. Запускать программу каждые 10 минут

```text
Название задачи: Проверка
Команда для запуска: notepad.exe
Тип расписания: 3
Интервал в минутах: 10
```

Такая задача будет запускаться каждые 10 минут, пока она включена и пока работает планировщик.

### Пример 4. Открыть файл `.docx`

Если нужно открыть документ или любой другой файл, нужно указывать полный путь к файлу в кавычках:

```text
"C:\Users\AKSI\Desktop\Документ.docx"
```

Если в пути есть пробелы, кавычки обязательны:

```text
"C:\Users\AKSI\Desktop\Мои документы\Отчет.docx"
```

Программа не ищет файл по всему компьютеру. Она запускает именно ту команду или путь, который ввел пользователь. Если написать только имя файла, программа сможет открыть его только в редком случае, когда этот файл находится в текущей рабочей папке запуска. Поэтому для файлов надежнее всегда указывать полный путь.

## Как работает программа

В программе есть структура `Task`, которая хранит данные одной задачи:

- `id` - номер задачи;
- `name` - название задачи;
- `command` - команда запуска;
- `type` - тип расписания;
- `nextRun` - ближайшее время запуска;
- `intervalMinutes` - интервал для повторяющихся задач;
- `active` - включена задача или выключена.

Когда пользователь запускает планировщик, программа входит в бесконечный цикл:

1. получает текущее время;
2. просматривает все активные задачи;
3. сравнивает текущее время с `nextRun`;
4. если время наступило, запускает команду;
5. пересчитывает следующее время запуска или выключает одноразовую задачу;
6. сохраняет изменения в `tasks.txt`;
7. ждет 5 секунд и повторяет проверку.

Команды запускаются через функцию `system()`. На Windows используется команда `start`, поэтому можно запускать не только `.exe`, но и файлы, которые открываются стандартными программами Windows.

## Файл tasks.txt

Все задачи сохраняются в файл: tasks.txt

Он создается автоматически рядом с программой. Благодаря этому задачи остаются после закрытия программы.

Редактировать `tasks.txt` вручную не рекомендуется, потому что программа ожидает определенный формат записи.

## Особенности

- Планировщик работает только пока открыта программа.
- Для запуска задач нужно выбрать пункт `5. Запустить планировщик`.
- Одноразовая задача после выполнения выключается.
- Ежедневная задача после выполнения переносится на следующий день.
- Интервальная задача после выполнения переносится на следующий интервал.
- Для запуска файлов лучше всегда указывать полный путь, например `C:\Users\AKSI\Desktop\file.docx`.
- Если путь к файлу содержит пробелы, его нужно брать в двойные кавычки.
